What is a Programming Strategy?

A strategy is a high-level plan for accomplishing a task, describing a series of steps or actions to take to accomplish a goal. Strategies are a common part of everyday life, describing steps with which to find an email in an email client, cook the perfect pizza, or tie your shoes. Programming strategies describe how software developers do similar everyday tasks, like different ways to debug a defect or go about software design.
In this study we are going through a set of tutorial steps which describe the syntax of alanguage for descibing strategies expert developers used in their everyday development life. This mixed-initiative language is called Roboto




Roboto Features

Roboto has several common features from imperative languages, including:

  1. Preconditions
  2. Statements
  3. Conditionals
  4. Actions
  5. Loops
  6. Variables

Precondition

Preconditions are the required resources, knowledge, skill, context , and any kind of requirements for using the strategy to fix the defect:
Context: Write your strategy for a specific context, such as a defect you debugged yesterday. Describe your application domain: embedded development in C, front end development in React, desktop Java application, etc.
Available resources: For your case, consider what resources are important to your strategy. Does it require that there be a test suite? A breakpoint debugger? A detailed log file?
Specific Programming Knowledge: What programming languages are required to be familiar with
Expertise level: what level of expertise the user of strategy should have


Statement

Each step in strategy definition is a statement describing what should the user do in next step

More description and clarification about the statement can be added before each statement after a # tag

Each statement is one of the following:

  1. ACTION
  2. CONDITIONAL
  3. LOOP
  4. CALL


Variable

Variables help users lower the burden of memorizing all the things.
Variable can be defined as a scenario , TODO step , list of steps, list of variables in a loop of steps, or any kind of variable the user need to keep for following steps’ use.

For instance, 'conflictedFiles' in the following statement is a variable which help user to keep it for further usages. This helps reduce memorization burden.



Conditional

A conditional asks the user to check if a condition is true in the world and then branch to different statements depending on the outcome.


Action

Actions describe something that the user should do in the world.


Call

A statement can call other strategies


Loops

Loop helps in tracking set of steps the user should take
Could be used to iterate over any set of similar actions the user should take
